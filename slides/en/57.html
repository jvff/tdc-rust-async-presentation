The methods for composing operations have been split off

  • The Future trait became simpler
  • It is more stable, with minimal (or no) changes between versions
  • The methods for composing operations were moved into a new FutureExt trait
    • map
    • map_err
    • and_then
    • or_else
    • then
    • flatten
    • etc.
  • New methods can be added without changing the Future trait
  • The trait can only be used if composing is needed
